Error:Execution failed for task ':app:javaPreCompileDebug'.

发表于 2018-02-28 | 分类于 问题解决

从 github 上克隆了学习项目后，build 时项目依赖的 ButterKnife 报如题错误。

解决办法

  1. 在 gradle 中添加如下代码

    android {
    compileSdkVersion 26
    buildToolsVersion '26.0.2'
    defaultConfig {
        applicationId "com.example.app"
        minSdkVersion 15
        targetSdkVersion 25
        versionCode 1
        versionName "1.0"
        testInstrumentationRunner "android.support.test.runner.AndroidJUnitRunner"

        //添加部分
        javaCompileOptions {
            annotationProcessorOptions {
                includeCompileClasspath true
            }
        }
    }
}

  2. 更新依赖的 ButterKnife 版本，并对源代码进行修改。

原因

因为 Android Studio 3.0 更新后对注解的使用与 ButterKnife 产生冲突所致。（具体解释可见Android studio 3.0 butterknife error #963）
由于很多项目大量使用 ButterKnife 注解，难以修改建议使用第一种方法。

参考：
