继承

  • 面向对象中，继承 就是一个类得到了另外一个类的成员变量和成员函数；
  • Java 当中只支持单继承，不允许多继承；（子类只有一个父类，而父类可以有多个子类。）
  • 使用继承可以减少重复代码量；
  • 继承的基本语法是在类名后使用 extends 关键字并加上想要继承的类名；

子类实例化过程

  • 子类不能继承父类的构造函数；
  • 但是，在子类的构造函数中必须调用父类的构造函数；（当没有调用时，编译器会自动添加 super()）
  • super 关键字用于调用父类中的函数和成员变量，类似 this；（减少重复代码）

复写

  • 在具有父子关系的两个类中，有两个函数它们的定义（返回值类型、函数名和参数列表）完全相同就是 复写；
  • 在子类的复写函数中，可以使用 super 调用父类中的成员函数，来消灭重复代码；

对象向上转型

向上转型 ：将子类的对象赋值给父类的引用，如下

//Person 是 Student 的父类。
Student s = new Student();
Person p = s;

语法特点：

  1. 转型后一个引用能够调用的成员，取决于这个引用的类型；（具体就是对象 p 不能调用 s 中独有的成员）
  2. 转型后一个引用调用的方法，取决于这个引用指向的对象；（具体就是对象 p 调用的方法是 s 中的复写方法）

对象向下转型

向下转型 ：将父类的对象赋值给子类的引用，如下

//Person 是 Student 的父类。
Student s1 = new Student();
Person p = s1; 
//向下转型前提为对象已经向上转型
Student s2 = (Student)p

语法特点：直接生成的父类对象，不能直接转型为子类。

抽象函数

只有函数的定义，没有函数体的函数被称为 抽象函数；

abstract void fun();
  • abstract 关键字修饰的函数为抽象函数；
  • 一个类当中如果有一个及以上的抽象函数则其必须被定义为 抽象类 ；

抽象类

使用 abstract 定义的类被称之为抽象类。

语法特征：

  1. 抽象类无法生成对象；
  2. 一个类包含抽象函数时，必须被声明为抽象类；
  3. 但如果没有抽象函数时，任然可以被声明为抽象类；

抽象类也可以称为 基类，且抽象类也具有构造函数（在继承后被调用）。

抽象类常用于定义通用父类（无具体方法实现的父类）时使用。

软件包

  • 软件包 可以简单的理解为类存放的文件夹；
  • 将类放置到一个包当中，需要使用 package + 包名 的声明；
  • 在使用命令行编译时，需要使用 -d + 目录参数；
  • 一个类的全名应该是 包名.类名；
  • 包名的命名规范：
    1. 要求包名所有的字母都要小写；
    2. 包名一般情况下是你的域名倒写；
  • 软件包为 Java 类提供了命名空间（不同包中可以重名）；
  • 在类文件声明包后，可以通过 import + 包名.*/包名.类名 的方式导入其他软件包；

访问权限

  1. public：公共权限，允许在不同包之间访问，没有任何限制；
  2. private：私有权限，仅限于当前类内调用，常用于修饰变量、函数和内部类；
  3. default：包级别访问权限（在不写修饰符时就是此权限），只能在同一包内访问；
  4. protected：受保护权限，功能同 default，但是只能修饰成员变量和成员函数，并且其修饰的成员能被子类从包外部调用。

如果子类和父类不在同一个包当中，子类可以继承到父类当中的 default 权限的成员变量和成员函数，但是权限不够无法使用。
public>protected>default>private
一个类和其成员的权限应该尽可能的小，以体现面向对象的封装性。

接口

接口就是标准，定义了接口就是定义了调用对象的标准。（使用接口需要实现，实现 implements 是一种特殊的继承。）参考资料

基本语法：

  1. 使用 interface 定义；
  2. 接口当中的方法都是抽象方法；
  3. 接口当中的方法都是 public 权限；
  4. 实现接口需要使用 implemnts 关键字；
  5. 一个类可以实现多个接口；
  6. 一个接口可以继承多个接口；
