异常

异常 ：中断了正常指令流的事件；

特点：

  1. 具体的某一个异常是一个对象，这些对象由 JDK 提供；
  2. 在遇到异常时，JVM 将创建一个异常对象；

分类：

  • Throwable 类（最基本的父类）
    • Exception 类 (Throwable 的直接子类) ：代表上面提到的 异常， 分为两大类；
      • RuntimeException 类 ：运行时异常（uncheck exception），编译器在编译时不会强制要求处理此异常；
      • …… ：编译时异常（check exception）,Java 编译器在编译时会强制要求处理此异常；
    • Error 类 (Throwable 的直接子类) ：虚拟机在运行时产生的 错误，一旦产生虚拟机就会关闭；

注意：Exception 本身也是 check exception。

处理：

try{

    //可能产生异常的代码

}catch(Exception e){
    e.printStackTrace() //打印异常的追踪信息
}finally{
    //代码 （异常的出口）
}

当处理后的代码产生异常时，进入 catch 中的代码运行，而 finally 中的代码无论是否出现异常，都会执行。

两个关键字

  • throw 后跟一个异常对象，用于抛出这个异常（JVM 得到这个对象后就会终止代码运行）,常用于自定义异常的抛出；
    • 如果抛出 check exception 必须对代码进行 try catch 或者声明异常；
  • throws后跟一个异常类型，用于在函数定义后声明异常，异常在调用此函数时再进行处理；

示例代码如下：

class User{
    private int age;

    public void setAge(int age) throws Exception {
        if（age < 0）{
            Exception e = new Exception("年龄不能为负数");
            throw e;
        }
        this.age = age;
    }
}

I/O 操作的目标

从数据源当中读取数据，以及将数据写入到数据目的地当中；

分类：

  • 第一种
    • 输入流：数据进入程序的“管道”；
    • 输出流：数据到达目的地的“管道”；
  • 第二种
    • 字节流：数据以字节为基础进行读取输出（mp3音乐文件常以字节流读取）；
    • 字符流：数据以字符为基础进行读取输出（文本文件常以字符流读取）；
  • 第三种
    • 节点流：在“装饰者”设计模式中真正处理数据的流；
    • 处理流：在节点流基础上对数据进行加工；

核心类：

  1. InputStream 字节输入流抽象类，常用子类为 FileInputStream，核心方法为 int read(byte[] b, int off, int len)；
  2. OutputStream 字节输出流抽象类，常用子类为 FileOutputStream，核心方法为 void write(byte[] b, int off, int len)；
  3. Reader 字符输入流抽象类，常用子类为 FileReader，核心方法为 int read(char[] c, int off, int len)；
  4. Writer 字节输出流抽象类，常用子类为 FileWriter，核心方法为 void write(char[] c, int off, int len);

byte[] b 用于存放字节流数据，int off 用于说明在数组第几位开始存放，int len 一次最多读取或输出多少数据；read 方法的返回值是这一次调用总共读取的字节数

注意：在读取大文件时需要循环读取；read 方法读取到末尾时会返回 -1；在 I/O 操作完成后，需要调用各对象的关闭方法，释放文件占用，且最好在 finally 代码块中执行（避免因代码终止为运行）；并且关闭流时，应注意关闭的顺序问题。 java中io各种流的关闭顺序

“装饰者”模式

BufferedReader字符输入处理流类，核心方法 public String readLine()（该方法一次性读取一行的数据，作为字符串返回），生成对象的方法是

BufferedReader br = new BufferedReader(new FileReader("文件地址"))

这里就用到了“装饰者”设计模式。

实例

让A公司的工人在工作前都说“你好“；先建立一个 Worker 接口，定义所有工人

public interface Worker {
    public void doSomeWork();
}

然后建立具体的工人类并实现 Worker 接口

//水管工
public class Plumber implements Worker{
    @Override
    public void doSomeWork() {
        System.out.println("修水管");
    }
}

//木匠
public class Carpenter implements Worker {
    @Override
    public void doSomeWork() {
        System.out.println("修门窗");
    }
}

再建立A公司工人类

public class AWorker implements Worker {

    public Worker worker;

    public AWorker(Worker worker){
        this.worker = worker;
    }

    @Override
    public void doSomeWork() {
        System.out.println("你好");
        worker.doSomeWork();
    }
}

注意 AWorker 将 Worker 对象作为成员变量，并在构造函数中赋值。然后在实现方法时调用传入 Worker 的方法；

public class TestDecorator {

    public static void main(String args []){
        //生成一个A公司水管工对象
        Plumber plumber = new Plumber();
        AWorker aWorker = new AWorker(plumber);

        aWorker.doSomeWork();

        //生成一个A公司的木匠对象
        Carpenter carpenter = new Carpenter();
        AWorker aWorker1 = new AWorker(carpenter);

        aWorker1.doSomeWork();
    }
}

运行主函数就解决了问题。

实例中的 AWorker 就称为“装饰者”，Carpenter 和 Plumber 为“被装饰者”，装饰者为被装饰者前面加上了“你好”，用三个类实现了四个类的功能（不用分别创建两个继承 Carpenter 和 Plumber 的A公司工人类），这就是装饰者模式。

注意：装饰者模式中，装饰者必须持有被装饰者的对象。

针对 I/O 流使用的装饰者模式，那么 FileReader 就是节点流，而 BufferedReader 就是处理流，处理流为节点流添加新的功能。
