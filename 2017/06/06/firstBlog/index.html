博客搭建过程记录与总结

从2017年3月22日到4月26日经过长时间与拖延症的斗争，我终于使用hexo + github pages的方法完成了个人博客的搭建。我搭建个人博客的起因是在进入 android 开发工作大半年以后，我猛然发现对于之前工作过程中遇到的各种问题和解决办法并没有有效的记录和总结，以致于在以后遇到同样的问题时，只能依靠脆弱的记忆以及在大量的项目代码中寻找这些低效率的办法。那么，现在就是体现个人博客价值的时候了，让我好好的理一理搭建博客过程中学习到的东西和遇到的坑。

Hexo + GitHub pages

首先要感谢这两篇博客：

  1. 我的博客是如何搭建的（github pages + HEXO + 域名绑定）
  2. GitHub Pages + Hexo 搭建博客

第一篇博客让我初步完成了博客搭建的流程，第二篇博客让我实现了博客的多终端管理。

接下来是我对搭建过程的总结：

1. github pages

只需要创建 github 仓库，注意仓库命名时需按照yuorusername.github.io的格式。

2. hexo

关于 hexo 安装、使用可参考 hexo官方文档
需要注意的是 hexo 和 github pages 的关联配置问题。在站点目录下的_config.yml文件中修改以下部分

# Deployment
    ## Docs: https://hexo.io/docs/deployment.html
    deploy:
    type: git
    repo: https://github.com/yourusername/yourusername.github.io.git
    branch: master

其中 repo 应为 github 仓库 https 地址。（由于初次搭建博客，所以我的主题使用的是第一篇博客中提到的next主题）

到此已经初步完成博客的搭建工作，可是我发现这样只能在一个终端操作站点，当更换电脑时就坑爹了。所以我又找到了第二篇关于在远程仓库保存站点目录的博客。

远程 git 仓库实现多终端同步管理博客

在搭建完成后，安装hexo-deployer-git插件将网站部署到github上后，可以看到远程仓库中只是上传了站点目录中 public 下的部分。因此当你在别的电脑上，就不可能通过 clone github pages 仓库来实现对
博客的管理。因此就需要：

1. 将站点目录上传 git 远程仓库

首先是创建远程 git 仓库，然后在站点目录下 git bash here 并使用以下命令

$ git init
$ git remote add [shortname] [url]

将站点目录与远程仓库关联，使用$ git remote -v可查看结果。
当关联成功后，就可以将本地内容上传了

$ git add .
$ git commit -m [提交说明]
$ git push origin master

我没有选择像第二篇博客中那样，使用 git 分支来保存，而是重新新建一个仓库。完全是因为对于我这样的小白来说，这样更加便于管理。

2. Git Submodule 链接主题仓库

在我将站点目录上传之后，我发现了一个问题。我使用的主题next是通过github下载的最新库（详见next主题的官方文档），在上传到远程仓库时只有一个空文件夹。在搜索后发现，如果在远程仓库中使用了第三方库需要通过git submodule关联到第三方库的github地址。因此，简单总结一下使用方法。

  1. 如何使用 Git Submodule
  2. 同步一个 fork

以上是参考的博客。

使用下面的 git 命令新增一个 Submodule 库。

$ git submodule add (repository) (directory)

注意repository指的是第三方库的地址，directory指用来存放代码的文件夹名称。命令git submodule status会告诉你这个 submodule 已经被注册了并且指向了哪一个commit.同时，使用git status可以查看库中的变化。接下来只要将改变提交并推送到远程仓库就可以了。而且，git 的常用命令都可以在这个库中使用，所以更新等操作也非常方便。更多的具体操作可参考第一篇博客。

总结

到此为止，我的博客基本搭建完毕。
